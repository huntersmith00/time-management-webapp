Project Overview

This project aims to help individuals effectively manage their time by providing an intuitive platform for scheduling tasks and events. Users will be able to organize their day, prioritize activities, and visualize their schedule in a clear, structured way. By simplifying the process of planning and tracking commitments, the application promotes better productivity and time management. Our goal is to offer a user-friendly tool that encourages individuals to stay on top of their personal and professional responsibilities.

Problem Description

Problem Domain

The problem domain for this project focuses on improving time management and organization for individuals juggling multiple responsibilities. Many people struggle to balance tasks, projects, and personal goals, which can lead to missed deadlines and decreased productivity. This application aims to provide users with a structured interface to create, track, and manage their goals, projects, and tasks, enabling them to allocate resources effectively and monitor progress. By enhancing the planning process through features like calendar view and priority settings, the application seeks to empower users to achieve their objectives more efficiently.

Need

There is a growing need for effective time management tools that can help individuals organize their lives and improve productivity. As the demands of work, school, and personal commitments increase, people require a solution that allows them to seamlessly manage their tasks and projects in one place. Current methods, such as traditional planners or fragmented digital solutions, often lead to confusion and disorganization, making it difficult to prioritize responsibilities. This project addresses this need by developing an integrated time-management application that empowers users to take control of their schedules, set clear goals, and track their progress, ultimately leading to improved efficiency and reduced stress.

Scope & Perspective

The scope of this project encompasses the development of a time-management database application designed to assist users in organizing their tasks, projects, and goals effectively. Key features will include a calendar for scheduling events and the ability to create, update, and delete tasks and projects. The application will also allow users to set priorities and track the completion status of their tasks, providing a holistic view of their time management.

From a user perspective, the application targets individuals looking for an efficient way to manage their personal and professional responsibilities. This includes students, professionals, and anyone seeking to enhance their productivity. The design will prioritize user experience, ensuring that the interface is intuitive and accessible, making it easy for users to navigate and utilize the application’s features effectively. By focusing on the needs of diverse users, this project aims to deliver a valuable tool that fosters better time management and goal achievement.

User Roles & Use Cases

Individuals seeking to better manage their time Needs: Manage schedules, deadlines, and tasks more effectively. Use Cases: Create and track tasks, set deadlines, and view schedules to meet important dates.

Task Management: Users can create, update, and delete tasks, projects, and goals with deadlines and priorities. Scheduling: A calendar feature allows users to view and manage their schedules. Progress Tracking: Users can track the completion status of tasks, projects, and goals.

Security & Privacy

User authentication is essential to verify the identity of individuals accessing the database, helping to protect sensitive information and ensure that only authorized users can interact with their data. This can be achieved by requiring users to create strong passwords or confirming their identity through other verification methods.

Relational Schemas

Key Variable Type Domain Constraint

Calendar Table

Key Variable Type Domain Constraint
FK UserID INT Created automatically by database Must exist in User Table
capacity INT makes sure we know how many tasks the user has currently, resets yearly must be less than 2000

Goal Table

Key Variable Type Domain Constraint
PK goalID INT makes it so we can track goals for each user Created automatically by system
goalName String describes the goal to be acheived must be less than 100 characters
capacity INT describes how many tasks are in the goal automatically managed by the system
priority ENUM describes the importance of the goal so they can be rank ordered must be one of the ENUMs that are specified
is_complete boolean so we can track completion of a goal must be true or false

Project Table

Key Variable Type Domain Constraint
PK projectID INT Tracks projects for a user automatically generated by system
projectName String Describes the project to be worked on Must be less than 100 characters
start INT When the task is supposed to begin Must be derived from the UNIX time
end INT When the task is supposed to end Must be derived from the UNIX time
priority ENUM Importance of the goal for rank ordering Must be one of the specified ENUMs
is_complete boolean Tracks completion of a goal Must be true or false

Task Table

Key Variable Type Domain Constraint
PK taskID INT Tracks tasks for a user Automatically generated by system
taskName String Describes the task to be achieved Must be less than 50 characters
start INT When the task is supposed to begin Must be derived from the UNIX universal time
end INT When the task is supposed to end Must be derived from the UNIX universal time
priority ENUM Importance of the goal for rank ordering Must be one of the specified ENUMs
is_complete boolean Tracks completion of a goal Must be true or false

Join Table

Key Variable Type Domain Constraint
FK taskID INT Tracks tasks for a user Must exist in Task table
FK userID INT Tracks which user is which Must exist in User Table
FK goalID INT Tracks goals for each user Must exist in Goal Table
FK projectID INT Tracks projects for a user Must exist in Project Table

Functional Dependencies & Normalization

User

Described by the functional dependency below. UserID closes firstName, lastName, time, and capacity because the Calendar entity depends on UserID and the rest of the attributes are just non-prime attributes in the User Table.

  • UserID -> ( name, capacity)

Goals

Described by the functional dependency below. GoalID closes goalName, capacity, and priority as these are the non-prime attributes in the Goal Table.

  • goalID -> (goalName, capacity, priority)

Tasks

Described by the functional dependency below. TaskID closes taskName, start, end, priority, and is_complete as these are the non-prime attributes in the Task Table.

  • TaskID -> (taskName, start, end, priority, is_complete)

Projects

Described by the functional dependency below. ProjectID closes projectName, start, end, priority, and is_complete because those are the non-prime attributes in the Project Table.

  • projectID -> (projectName, start, end, priority, is_complete)

Discussion of Normalization

This relational schema is in BCNF because it is atomic, there are no partial dependencies as each table only has one key being the primary key, there are no transistive dependencies, and each primary key is on the leftmost column and joined by a join table.

Specific Queries

  1. List all users:
    \(\pi_{userID, name}(User)\)

  2. Get all tasks for a specific user:
    \(\sigma_{userID = 'x'}(Task)\)

  3. Count total tasks for each user:
    \(\gamma_{userID, \text{COUNT}(taskID) \rightarrow total\_tasks}(Task)\)

  4. Retrieve tasks by priority:
    \(\sigma_{priority = 'High'}(Task)\)

  5. Find incomplete tasks:
    \(\sigma_{is\_complete = FALSE}(Task)\)

  6. Get projects for a specific user:
    \(\sigma_{userID = 'x'}(Project)\)

  7. List goals for a user:
    \(\sigma_{userID = 'x'}(Goal)\)

  8. Get tasks associated with a specific project:
    \(\sigma_{projectID = 'x'}(Task)\)

  9. Find all tasks in a project with a specific priority:
    \(\sigma_{priority = 'High' \land projectID = 'x'}(Task)\)

  10. Get tasks with deadlines within a range:
    \(\sigma_{start \leq 'start\_date' \land end \geq 'end\_date'}(Task)\)

  11. List all projects for a goal:
    \(\sigma_{goalID = 'x'}(Project)\)

  12. Get the name of all tasks and the corresponding project (join):
    \(\pi_{task\_name, project\_name}(Task \bowtie Project)\)

  13. Find the task count for each project:
    \(\gamma_{projectID, \text{COUNT}(taskID) \rightarrow task\_count}(Task)\)

  14. Get all high-priority tasks that are incomplete:
    \(\sigma_{priority = 'High' \land is\_complete = FALSE}(Task)\)

  15. Get the list of projects and their completion status:
    \(\pi_{project\_name, is\_complete}(Project)\)

  16. Retrieve tasks for a user that are part of a specific goal:
    \(\sigma_{userID = 'x' \land goalID = 'y'}(Task)\)

  17. Find all tasks that are overdue (assuming “today” is a known constant):
    \(\sigma_{end < 'today'}(Task)\)

  18. Get all goals with a certain priority:
    \(\sigma_{priority = 'High'}(Goal)\)

  19. Retrieve incomplete projects:
    \(\sigma_{is\_complete = FALSE}(Project)\)

  20. List tasks, their priority, and completion status:
    \(\pi_{task\_name, priority, is\_complete}(Task)\)
